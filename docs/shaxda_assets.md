# Shaxda Assets

This document records the Phase 1 E1 asset direction and source notes. E1 keeps
assets ready for later polish without adding gameplay audio playback, board
animation polish, or service-worker behavior.

## Visual Direction

The board should feel like a carved wooden Shaxda board: warm, simple, and easy
to scan on mobile. V1.0 should use the existing CSS and SVG direction rather
than heavy raster textures.

- Board texture: use warm wood colors from `--color-board-*` in
  `web/src/app.css`, with `#6e4327` as the main board brown and `#f8f1e8` as the
  light surface color from `packages/ui/src/index.ts`.
- Board lines: keep carved-looking dark strokes with rounded ends, matching the
  current SVG board in `web/src/lib/components/Board.svelte`.
- Pieces: use raised wooden discs. Player A uses a light cream-to-brown radial
  gradient; Player B uses a darker brown radial gradient. Both keep a subtle
  drop shadow.
- Icons: SVG remains the source of truth. The icon direction is a brown square
  with a cream board motif. The maskable icon keeps extra safe-area padding.
- Open Graph: SVG remains the source of truth. The OG direction is a 1200x630
  warm board scene with the Shaxda wordmark and Somali supporting text.

## Sound Direction

The six E1 sound files are short, generated WAV cues. They are intentionally
simple placeholders for V1.0 interaction feedback, not background music.

| Cue       | Direction                                    |
| --------- | -------------------------------------------- |
| `place`   | Soft wooden tock with low quick decay        |
| `move`    | Lighter higher tick                          |
| `jare`    | Pleasant rising two-note chime               |
| `capture` | Firmer thud/pop with a slightly longer decay |
| `invalid` | Short low dull warning tone                  |
| `win`     | Brief ascending three-note flourish          |
